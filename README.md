# Определение популярности геолокации для размещения банкомата

## О проекте

Цель проета — разработать модель машинного обучения, которая предсказывает потенциальную популярность банкомата в зависимости от характеристик его окружения. На основе предсказаний модели будет создан FastAPI сервис для определения перспективных локаций для размещения банкоматов.

**Ключевые этапы:**
- Сбор и обработка данных из OpenStreetMap и API Яндекс.Карт.
- Разведочный анализ данных (EDA) и feature engineering.
- Обучение и сравнение различных ML и DL моделей.
- Создание FastAPI сервиса для демонстрации результатов.

## План работы

### Этап 1: Сбор данных и Разведочный анализ данных (EDA)

**Цель:** Получить обогащенный датасет для обучения и провести его первичный анализ.

**Задачи:**
1.  **Исходные данные:** Анализ имеющегося датасета с координатами банкоматов и метриками популярности.
2.  **Сбор фич:**
    - Использование **OpenStreetMap** для сбора информации об объектах инфраструктуры (кафе, магазины, метро, остановки, бизнес-центры и т.д.) в радиусе вокруг каждого банкомата.
    - Использование **API Яндекс.Карт** (при необходимости) для получения дополнительных данных (например, пешеходный и автомобильный трафик).
3.  **Предобработка данных:** Очистка данных, обработка пропусков, кодирование категориальных признаков.
4.  **EDA:** Визуализация распределения признаков и целевой переменной, анализ корреляций, выявление аномалий и закономерностей.
5.  **Формирование финального датасета:** Создание таблицы "объект-признак" для передачи в модели.

---

### Этап 2: Линейные модели Machine Learning

**Цель:** Создать и оценить базовые линейные модели.

**Задачи:**
1.  Подготовка данных для обучения: масштабирование признаков, разделение на train/valid/test выборки.
2.  Обучение и валидация линейных моделей (например, **Linear Regression**, **Ridge**, **Lasso**).
3.  Оценка качества моделей с помощью адекватных метрик (MAE, MSE, R2).
4.  Анализ важности признаков в линейных моделях.

---

### Этап 3: Нелинейные модели ML, Feature Engineering и выбор лучшего ML-решения

**Цель:** Улучшить качество предсказания за счет более сложных моделей и генерации новых признаков.

**Задачи:**
1.  **Feature Engineering:** Создание новых признаков (например, полиномиальные, агрегаты по районам, расстояние до ближайших ключевых объектов).
2.  Обучение и валидация ансамблевых моделей (например, **Random Forest**, **Gradient Boosting (CatBoost, XGBoost, LightGBM)**).
3.  Сравнение качества всех обученных ML-моделей между собой.
4.  **Подбор гиперпараметров** для лучших моделей.
5.  **Выбор чемпиона** среди классических ML-подходов.

---

### Этап 4: Создание сервиса на FastAPI с лучшей ML-моделью

**Цель:** Разработать интерактивное веб-приложение для демонстрации работы модели.

**Задачи:**
1.  Сохранение обученной модели-чемпиона.
2.  Разработка **FastAPI сервиса**, включающего:
    - Возможность сделать предсказание по координатам;
    - Просмотра истории и статистики запросов.
3.  Деплой приложения для демонстрации результатов.

---

### Этап 5: Deep Learning модели

**Цель:** Исследовать возможность улучшения результатов с помощью нейронных сетей.

**Задачи:**
1.  Подготовка данных для нейронных сетей (дополнительная нормализация).
2.  Разработка и обучение нескольких архитектур **полносвязных нейронных сетей (FNN)**.
3.  Эксперименты с различными функциями активации, оптимизаторами и техниками регуляризации (Dropout, BatchNorm).
4.  Сравнение качества лучшей DL-модели с чемпионом этапа ML.

---

### Этап 6: Доработка и финализация

**Цель:** Улучшение решения по итогам обратной связи и выбор финальной модели.

**Задачи:**
1.  **Тюнинг моделей:** Более глубокий подбор гиперпараметров для лучших моделей.
2. **Интеграция с MLflow:** Ведение экспериментов и сравнение моделей с помощью MLflow.
3.  **Финальное сравнение:** Выбор лучшей модели overall (ML vs DL) по качеству, скорости работы и надежности.
4.  Интеграция финальной модели в FastAPI-сервис.
5.  Подготовка финального отчета и презентации по проекту.
